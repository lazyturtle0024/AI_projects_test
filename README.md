明天练习的核心能力是：

Clone → Run → Modify → Push

具体流程：

1. 找到一个简单的 GitHub 项目。
2. Clone 或下载到本地。
3. 按照 README 在本地跑起来。
4. 对代码做一个很小的修改。
5. 保存修改。
6. Commit。
7. Push 到我自己的 GitHub 项目里。
8. 写一个简单 closure，记录这个项目学到了什么。

## 今天学到的核心理解

GitHub 不是普通网盘。

它更像是一个项目仓库 + 版本管理系统 + 未来项目的入口。

Git 负责版本管理，GitHub 负责把项目放到云端，方便保存、同步、管理、展示和协作。

Repo 是一个带历史记录的项目仓库。

Commit 是一次有说明的版本记录。

Branch 可以理解成草稿区或实验分支。

Push 是把本地修改推送到 GitHub。

Pull 是从 GitHub 拉取最新版本到本地。

GitHub Pages 可以把 HTML 文件发布成网页。
